# 文件说明：这些项目文件能干嘛

这个文件用于把项目里的英文文件名翻译成中文用途说明。以后如果你看到生成结果里只有英文文件名，可以先看这里。

## 文件对照表

| 英文文件名 | 中文备注 | 你可以用它干嘛 |
|---|---|---|
| `collaboration-guidelines.md` | 协作规范 | 记录我们怎么讨论公式、怎么验证、我应该如何主动提醒你、如何避免过拟合。换新对话时，可以让我先读取它，恢复这套协作方式。 |
| `current-checkpoint.md` | 当前断点 | 记录我们现在讨论到哪、最近确认了什么、下一步该继续什么。上下文很长或中途插问后，用它恢复主线。 |
| `initial-formula-original.txt` | 初始公式原版 | 保存你最开始发给我的完整原公式。后续做“原公式 vs 整理版/新版公式”对比时用它作为基准。 |
| `formula-discussion-summary.md` | 公式讨论摘要 | 记录已经确认的公式修改、当前公式结构、重要结论和待讨论事项。它是主公式讨论的核心记录。 |
| `project-index.md` | 项目索引 | 说明整个项目有哪些文件、每个文件负责什么、当前处于什么阶段。相当于项目目录说明。 |
| `pending-verification-list.md` | 待验证清单 | 记录所有后续要验证/回测的问题，比如某个参数是否偏慢、风险是否误杀、信号是否太少。 |
| `validation-indicator-plan.md` | 验证副图计划 | 后续设计“验证指标/诊断副图”时使用，用来帮助判断某根K线为什么出信号、为什么不出信号。 |
| `backtest-plan.md` | 回测计划 | 后续你导出 Python 数据后，用来设计回测统计口径，例如3/5/10/20根K线后的表现、胜率、回撤等。 |
| `formula-changelog.md` | 公式版本记录 | 记录每一版公式改了什么、为什么改、影响哪些信号、可能有什么副作用。 |

## 最常用的三个文件

如果你只想看最关键的，优先看这三个：

1. `current-checkpoint.md`：看我们当前聊到哪里。
2. `formula-discussion-summary.md`：看公式已经确认了哪些内容。
3. `pending-verification-list.md`：看后续要验证哪些问题。

## 换新对话时怎么用

如果以后换了新对话，你可以直接说：

```text
请先读取 文件说明-中文索引.md、current-checkpoint.md、collaboration-guidelines.md 和 formula-discussion-summary.md，恢复我们之前的公式讨论状态。
```

这样我就能更快回到当前项目，不需要你重新解释很多背景。

## 为什么暂时不直接改英文文件名

英文文件名更适合工具、脚本、链接和长期维护。现在很多文件之间已经互相引用，如果直接改名，容易造成引用断裂。

所以当前做法是：

- 文件名继续保持英文，保证稳定；
- 用这个中文索引告诉你每个文件的中文含义和用途；
- 后续如果你希望，我们可以再统一改成“中文备注 + 英文名”的命名方式，例如 `当前断点-current-checkpoint.md`。

## 初始公式原版在哪里

文件名：`initial-formula-original.txt`

中文含义：初始公式原版。

用途：保存你最开始发给我的完整公式，后续用于和整理版公式、新版公式做逐项对比。

如果右侧“生成结果”里暂时看不到这个文件，优先以这个索引记录为准；文件实际已经在项目目录里。
