# 公式优化项目索引

本项目用于持续迭代东方财富通公式，包括主公式讨论、协作规范、验证副图、回测计划、待验证问题和版本变更记录。

## 文件说明

- `文件说明-中文索引.md`  
  中文文件说明。把英文文件名翻译成中文用途，说明每个文件能干嘛、什么时候用。

- `initial-formula-original.txt`  
  初始公式原版。保存用户最开始发来的完整公式，后续用于和整理版、新版公式做逐项对比。

- `formula-discussion-summary.md`  
  主公式讨论摘要。记录已确认修改、当前公式结构、待讨论问题和待验证项。

- `collaboration-guidelines.md`  
  协作规范。记录讨论方法、优化原则和如何更好发挥助手能力。

- `formula-changelog.md`  
  公式版本记录。后续每形成一个阶段版本，就记录改动、原因、影响和风险。

- `validation-indicator-plan.md`  
  验证副图计划。等用户说“开始验证指标讨论”后重点维护。

- `backtest-plan.md`  
  回测计划。记录后续 Python 回测的统计口径，等主公式和验证指标稳定后维护。

- `pending-verification-list.md`  
  待验证清单。记录所有用户和助手提出的潜在问题、参数疑点、不利影响和后续观察项。

- `current-checkpoint.md`  
  当前断点。记录主线停在哪里、最近已确认什么、当前待确认什么、插问处理后应回到哪里。

## 当前阶段

当前处于主公式逻辑讨论阶段。

暂定流程：

1. 完成主公式逐模块讨论。
2. 整理确认版主公式。
3. 设计验证/诊断副图。
4. 用验证副图人工检查信号合理性。
5. 设计并执行回测。
6. 根据验证和回测结果做第二轮优化。

## 助手主动维护原则

用户不需要记住所有可用能力。后续由助手主动判断何时维护或使用以下能力：

- 更新讨论摘要；
- 维护版本变更记录；
- 整理待验证问题清单；
- 生成原公式/新公式/影响对比；
- 设计验证副图；
- 设计错误定位/条件断点表；
- 设计 Python 回测脚本；
- 整理和重构公式；
- 做阶段总结和上下文恢复。
