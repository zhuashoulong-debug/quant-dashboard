# 东方财富通公式讨论记录

本记录保存截至当前已经确认的主要讨论结论，后续继续讨论时以此为阶段基准。

## 总体原则

- 公式目标偏向“所有强势突破都提示，但靠险、败、慎、滞控制风险”。
- 不盲目继续加复杂条件，优先做减法和分层回测。
- `确` 不作为主要首买点，更适合作为高质启动后的持有、加仓、确认信号。
- 涉及公式时，先给公式，再用中文解释公式含义。
- 后续优化不能为了适配某一只股票、某一段历史走势而过度调整公式；不能追求“过去看起来特别准”而牺牲客观逻辑。所有修改都必须逻辑顺畅、合理、周延，且程度适中。
- 对个例问题的处理原则：先判断是否代表一类稳定现象；若只是个别走势，不为其专门加复杂条件。优先用通用逻辑、分层提示或参数微调解决。

## 已确认的基础优化

### 趋势与带宽参考周期

当前暂时保留：

```pascal
TN:=IF(周线,26,IF(月线,24,60));
TM:=IF(周线,5,IF(月线,3,10));

WM:=IF(周线,26,IF(月线,24,60));
WRP:=0.85;
```

含义：

- `TN` 是长线趋势均线周期，日线60、周线26、月线24。
- `TM` 是判断长线方向的对比周期，日线10、周线5、月线3。
- `WM` 是带宽历史参考周期，日线60、周线26、月线24。
- `WRP=0.85` 表示当前带宽达到阶段高宽的85%以上时，视为接近高宽。

当前状态：

- 暂时接受，不改主公式。
- 不是因为这些参数一定合理，而是因为它们属于系统级参数，牵动趋势、带宽、迟发、首扩和风险模块。
- 后续作为专项验证项：观察日线趋势是否偏慢、带宽参考是否滞后、`WRP=0.85` 是否过早触发 `宽近高` 并压制迟发、首扩、平台等信号。

### 显示许可

新增最终显示层过滤：

```pascal
有效周期:=日线 OR 周线 OR 月线;
足够K:=BARSCOUNT(C)>IF(周线,80,IF(月线,48,120));
显示许可:=有效周期 AND 足够K;
```

含义：

- 只在日线、周线、月线显示信号。
- 历史K线数量不足时不显示最终信号。
- 只用于最终显示层，不改变中间逻辑。

### 成交额防空

原来：

```pascal
AMT:=AMOUNT;
均额:=MA(AMT,VN);
温额:=AMT>=均额;
放额:=AMT>=均额*VP;
巨额:=AMT>=均额*BVP;
```

改为：

```pascal
AMT0:=IF(AMOUNT>0,AMOUNT,VOL*C*100);
均额:=MA(AMT0,VN);
有额:=AMT0>0 AND 均额>0;

温额:=有额 AND AMT0>=均额;
放额:=有额 AND AMT0>=均额*VP;
缩额:=有额 AND AMT0<=均额*0.8;
巨额:=有额 AND AMT0>=均额*BVP;
```

影响：

- 对普通A股影响较小。
- 对 `AMOUNT` 异常、指数、ETF、数据缺失场景更稳。

### 成交额窗口与阈值

当前暂时保留：

```pascal
VN:=IF(周线,13,IF(月线,12,20));
VP:=1.2;
BVP:=1.8;
```

含义：

- `VN` 是成交额均线周期：日线20天、周线13周、月线12个月。
- `VP=1.2` 表示当前成交额达到均额的1.2倍以上，算放额。
- `BVP=1.8` 表示当前成交额达到均额的1.8倍以上，算巨额。

当前状态：

- 暂时接受，不改主公式。
- 但这是高优先级验证项，不是低风险参数。
- 原因是 `VP / BVP` 同时影响启动、确认和风险模块，贸然修改会让多类信号同时变化。

后续验证重点：

- `VP=1.2` 是否让放额太容易成立。
- `BVP=1.8` 是否能有效识别真正巨额风险。
- 巨额长上、极上险等风险是否太敏感或太迟钝。
- 不同市值、不同成交额波动股票是否表现差异明显。
- 验证副图必须显示 `AMT0`、`均额`、`额比:=AMT0/MAX(均额,0.0001)`、`温额`、`放额`、`巨额`，以及当前额比在近期成交额中的相对位置。

### K线质量参数

当前暂时保留：

```pascal
BP:=0.45;
YP:=0.45;
```

含义：

- `BP=0.45` 用来判断实体强度，实体占整根K线比例达到45%以上，才算实体较强。
- `YP=0.45` 用来判断影线长度，上影线占整根K线比例达到45%以上，算长上影。

对应公式：

```pascal
阳强:=C>O AND 实占>=BP;
长上:=上影>=YP;
```

当前状态：

- 暂时接受，不改主公式。
- `阳强` 继续作为高质量突破的主要K线质量标准。
- `长上` 继续用于过滤明显冲高回落和识别风险。
- `中阳无长上` 暂不加入主公式，进入验证副图和待验证清单，用来判断 `BP=0.45` 是否误伤温和有效启动。

后续验证重点：

- `BP=0.45` 是否让启动或确认信号过少。
- `中阳无长上` 是否有明显后续表现。
- `YP=0.45` 是否误伤强势换手突破。
- 长上影风险是否过早压制强势票。

### 结构突破与失败窗口

当前暂时保留：

```pascal
BRN:=IF(周线,13,IF(月线,12,20));
FN:=IF(周线,3,IF(月线,2,5));
```

含义：

- `BRN` 是结构突破回看窗口，日线20天、周线13周、月线12个月。
- `构上` 表示当前收盘价突破过去 `BRN` 根K线的实体高点。
- `构下` 表示当前收盘价跌破过去 `BRN` 根K线的实体低点。
- `FN` 是启动失败观察窗口，日线5天、周线3周、月线2个月。

当前状态：

- 暂时接受，不改主公式。
- `BRN` 的周期有清楚市场含义，日线约一个交易月、周线约一季度、月线约一年。
- `FN` 作为启动后失败观察窗口暂时合理。
- `FN=5` 是否对日线震荡型启动过早标败，进入高优先级验证。

后续验证重点：

- `BRN` 是否让 `构上` 太难或太容易成立。
- `FN` 是否让 `上败` 太早提示，误伤正常回踩。
- 对比原版 `上败原` 与候选 `上败弱化:=PCTB<0.7 OR (C<MID AND C<短线)`。

### 黄色冷却与新鲜度参数

当前暂时保留：

```pascal
黄色冷却N:=IF(周线,4,IF(月线,3,8));
黄色趋势段N:=IF(周线,8,IF(月线,5,15));
黄新鲜N:=IF(周线,5,IF(月线,3,10));
```

含义：

- `黄色冷却N` 是短冷却窗口，防止近期已经出现上轨信号后又重复显示普通黄。
- `黄色趋势段N` 是趋势段窗口，判断是否已经处于一段强趋势运行中。
- `黄新鲜N` 是新鲜度窗口，判断近期是否已经出现强推、启、速、冲、优质黄等信号。

当前状态：

- 暂时接受，不改主公式。
- 黄色信号质量低于启、蓝、速、冲，适当冷却是合理的。
- 但该组属于中高优先级验证项，因为多个冷却叠加后可能压制有效二次黄信号。

后续验证重点：

- 黄色信号是否被压得太少。
- 被压掉的黄色是否存在有效二次突破。
- `黄色冷却N=8` 是否过长。
- `黄色趋势段N=15` 是否过度压制强趋势中的新机会。
- `黄新鲜N=10` 是否和趋势段过滤重复过度。
- 验证副图需要显示 `冷却占用信号原`、`近上轨信号`、`强趋势运行`、`黄色显示许可`、`黄色新鲜许可` 等关键链路。

### 败速冲冷却参数

当前暂时保留：

```pascal
败冷却N:=IF(周线,3,IF(月线,2,5));
速冷却N:=IF(周线,4,IF(月线,3,8));
速后N:=IF(周线,5,IF(月线,3,12));
冲冷却N:=IF(周线,4,IF(月线,3,8));
```

含义：

- `败冷却N` 用于防止失败信号连续重复显示。
- `速冷却N` 用于防止 `速` 信号短期重复。
- `速后N` 用于判断是否处于 `速` 信号后的运行段。
- `冲冷却N` 用于防止 `冲` 信号短期重复。

当前状态：

- 暂时接受，不改主公式。
- 这些参数主要控制重复显示，不直接决定源头信号是否成立。
- `速后N=12` 是否过长、是否压制有效 `冲`，进入后续验证。
- `速后强运行` 作为候选优化进入待验证清单，暂不进入主公式。

后续验证重点：

- `速后N=12` 是否过度压制 `冲`。
- `冲冷却N=8` 是否压制有效二次平台突破。
- `败冷却N=5` 是否让失败提示太少。
- `速冷却N=8` 是否让二次加速被漏掉。
- 对比原版 `速后运行` 和候选 `速后强运行`，观察候选版放出的 `冲` 是否有效。

## 公式主体讨论

### 布林带与 PCTB

当前保留不改：

```pascal
MID:=MA(C,BBN);
TMP:=STD(C,BBN);
BUP:=MID+P*TMP;
BDN:=MID-P*TMP;

PCTB:(C-BDN)/MAX(BUP-BDN,0.0001),COLORYELLOW,LINETHICK2;
```

含义：

- `MID` 是布林中轨，即 `BBN` 周期收盘价均线。
- `TMP` 是 `BBN` 周期收盘价标准差。
- `BUP / BDN` 是2倍标准差布林上轨和下轨。
- `PCTB` 是当前收盘价在布林上下轨之间的相对位置。

当前状态：

- 接受，保留不改。
- `PCTB` 继续作为全公式核心强弱坐标。
- 后续上穿、极上、确认强区、布林滞后保护、风险判断等模块继续围绕 `PCTB` 展开。

后续验证副图要求：

- 显示 `MID`、`BUP`、`BDN`、`BUP-BDN`、`PCTB`、`PCTB` 今日值与昨日值。
- 显示 `PCTB>1`、`PCTB>=1+深限`、`PCTB<0.7`、`PCTB<0.5` 等关键状态。

### Keltner 通道与挤压

Keltner 通道与挤压定义保留：

```pascal
TRV:=MAX(MAX(H-L,ABS(H-REF(C,1))),ABS(L-REF(C,1)));
ATRV:=MA(TRV,KCN);
KCU:=MID+KCP*ATRV;
KCD:=MID-KCP*ATRV;

挤压:=BUP<KCU AND BDN>KCD;
```

新版近挤与效挤：

```pascal
近挤:=COUNT(挤压,2)>=1;

效挤:=IF(日线,
COUNT(挤压,5)>=2 AND BARSLAST(挤压)<=2,
COUNT(挤压,SQN)>=2);
```

含义：

- `TRV` 是真实波幅，包含跳空缺口。
- `ATRV` 是 `KCN` 周期真实波幅均值。
- `KCU / KCD` 是以 `MID` 为中轨、上下1.5倍ATR的 Keltner 通道。
- `挤压` 表示布林带完全收进 Keltner 通道内部。
- 新版 `近挤` 只代表最近2根内出现挤压，负责刚挤压后的快速突破背景。
- 新版 `效挤` 日线要求最近5天至少2次挤压，且最近一次挤压不超过2天；周线/月线仍按 `SQN` 至少2次挤压。

当前状态：

- Keltner 通道和挤压定义接受保留。
- `近挤 / 效挤` 接受采用新版逻辑。
- 因为 `近挤` 缩短，`迟发挤压背景` 的设计特别重要，负责补足挤压后第3天、第5天、第8天等延迟突破，同时避免错放已经扩张后的旧挤压。

后续验证重点：

- 新版 `近挤` 是否减少太多普通突破背景。
- 新版 `效挤` 是否足够识别密集挤压。
- `迟发挤压背景` 是否补到合理延迟突破。
- `迟发挤压背景` 是否错放已经扩张后的旧挤压。
- 验证副图需要显示原版/新版 `近挤`、原版/新版 `效挤`、`BARSLAST(挤压)`、`迟发挤压背景`、`有效挤压`。

### 布林带宽度状态

当前保留：

```pascal
带宽:=(BUP-BDN)/MAX(MID,0.0001);

长均宽:=MA(带宽,WM);
短均宽:=MA(带宽,BBN);

长高宽:=HHV(带宽,WM);
短高宽:=HHV(带宽,BBN);

长低宽:=LLV(带宽,WM);
短低宽:=LLV(带宽,BBN);
```

含义：

- `带宽` 是布林上下轨宽度相对中轨价格的比例。
- `长均宽 / 短均宽` 是长短窗口平均带宽。
- `长高宽 / 短高宽` 是长短窗口最高带宽。
- `长低宽 / 短低宽` 是长短窗口最低带宽。

当前状态：

- 接受保留，不改主公式。
- 这些变量是后续高宽、低宽、收缩、迟发、持宽判断的基础数据层。
- 需要注意：高宽/低宽可能受极端值影响，`WM` 长窗口可能对波动状态切换反应偏慢。

有效参考兜底保留：

```pascal
有效宽均:=IF(长均宽>0.001,长均宽,IF(短均宽>0.001,短均宽,带宽));
有效高宽:=IF(长高宽>0.001,长高宽,IF(短高宽>0.001,短高宽,带宽));
有效低宽:=IF(长低宽>0.001,长低宽,IF(短低宽>0.001,短低宽,带宽));
```

含义与状态：

- 这层主要是防异常兜底，避免历史不足或参考值过小时导致计算异常。
- 它能防止异常值过小，但不能解决极端高低宽污染和波动切换滞后。
- 暂时接受，后续验证副图必须显示长短参考和有效参考。

后续验证重点：

- `有效宽均` 是否长期跟不上波动状态变化。
- `有效高宽` 是否被历史极端高宽拉高。
- `有效低宽` 是否被历史极端低宽拉低。
- 是否需要后续设计短长参考自适应切换，例如观察短均宽和长均宽差异。

前有效参考与宽比保留：

```pascal
前有效宽均:=IF(REF(有效宽均,1)>0.001,REF(有效宽均,1),MAX(REF(带宽,1),0.0001));
前有效高宽:=IF(REF(有效高宽,1)>0.001,REF(有效高宽,1),MAX(REF(带宽,1),0.0001));
前有效低宽:=IF(REF(有效低宽,1)>0.001,REF(有效低宽,1),MAX(REF(带宽,1),0.0001));

当前宽比:=带宽/MAX(有效宽均,0.0001);
前宽比:=REF(带宽,1)/MAX(前有效宽均,0.0001);
```

含义与状态：

- `前有效宽均 / 前有效高宽 / 前有效低宽` 是上一根K线时的有效参考，用于判断启动前带宽环境。
- `当前宽比` 是当前带宽相对当前有效均宽的比例。
- `前宽比` 是上一根带宽相对上一根有效均宽的比例。
- 计算逻辑接受保留。
- 已确认整理版统一命名：删除 `前宽比2`，只保留 `前宽比`。

高宽判断保留：

```pascal
带宽绝高线:=IF(周线,0.55,IF(月线,0.80,0.35));
带宽相高线:=IF(周线,1.25,IF(月线,1.30,1.35));

带宽绝高:=带宽>带宽绝高线;
前带宽绝高:=REF(带宽,1)>带宽绝高线;

带宽相高:=当前宽比>带宽相高线 OR 前宽比>带宽相高线;
```

含义与状态：

- `带宽绝高` 判断带宽本身是否已经达到绝对高宽。
- `带宽相高` 判断当前或上一根带宽是否相对有效均宽明显偏高。
- 接受保留，后续验证。
- 注意这里的绝对高宽线是全局高宽判断，和迟发挤压里的动态迟发绝宽线不是一回事。

后续验证重点：

- 日线绝对高宽0.35是否过高，导致风险或迟发过滤偏晚。
- 日线相对高宽1.35是否太松。
- 周线1.25是否太敏感。
- 月线1.30是否合理。
- 当前+上一根是否足够，是否存在上上根高宽仍应过滤的情况。

持宽判断保留：

```pascal
宽近高:=带宽>=有效高宽*WRP;

宽速N:=IF(周线,3,IF(月线,2,5));
宽速:=带宽-REF(带宽,宽速N);
宽速率:=宽速/MAX(REF(带宽,宽速N),0.0001);

微扩阈:=IF(周线,0.05,IF(月线,0.06,0.04));

硬高宽:=带宽绝高 OR 前带宽绝高 OR 带宽相高;
软持宽:=宽近高 AND 宽速率<=微扩阈;

持宽:=硬高宽 OR 软持宽;
```

含义与状态：

- `宽近高` 表示当前带宽达到有效高宽的85%以上，接近阶段高宽。
- `宽速率` 表示带宽相对前一段时间的扩张/收缩幅度。
- `硬高宽` 抓明显高宽状态。
- `软持宽` 抓“接近阶段高宽，并且没有继续明显扩张”的高宽维持状态。
- `持宽` 是硬高宽和软持宽的综合。
- 接受保留，列为高优先级验证项。

后续验证重点：

### 收缩识别参数

当前暂时保留：


- `低位均比 / 低位低比` 是否需要按二年位动态分档。

```pascal

带宽低点近:=REF(LLV(带宽,缩窄N),1);

压缩低位:=带宽低点近<前有效宽均*低位均比
```

含义与状态：

后续验证重点：

- `带宽高点前` 是否被极端高宽污染，导致收缩幅度过容易成立。
- `带宽低点近` 是否被单日极低值影响，导致压缩低位过容易成立。
- `压缩低位` 的 OR 逻辑是否偏宽。

- 后续验证副图加入候选观察：

```pascal
早期趋势健康:=C>短线
AND C>MID
AND C>REF(HHV(实体高,滞后强N),1)
AND 阳强;

趋势健康扩展:=趋势健康 OR 早期趋势健康;
```

- 该候选暂不进主公式，用于观察早期强启动是否因 `短向>0` 未满足而缺少布林滞后保护。

### 价格继续强

当前讨论状态：

```pascal
价格继续强:=C>REF(C,1)
OR 构上
OR C>REF(HHV(实体高,滞后强N),1);
```

- 接受保留。
- 作用：判断价格本身是否还在继续推进。
- 三条路径分别是：收盘高于上一根、结构突破、收盘突破过去一段时间实体高点。
- 它是布林滞后保护的必要前提之一，避免走势已经不推进时仍被保护。

### 布林跟随扩张

当前讨论状态：

```pascal
布林跟随扩张:=BUP>REF(BUP,1)
AND MID>REF(MID,1);
```

- 接受保留。
- 作用：判断布林上轨和中轨是否跟随趋势上移。
- 如果价格仍强、布林带也上移，那么 `PCTB` 回落可能只是布林带扩张造成的表面变弱。

### PCTB伪弱

当前讨论状态：

```pascal
PCTB伪弱:=趋势健康
AND 价格继续强
AND 布林跟随扩张
AND PCTB<REF(PCTB,1)
AND PCTB>0.75
AND NOT(长上);
```

- 接受保留。
- 作用：识别 PCTB 虽然下降，但价格、趋势和布林带仍健康上行的“伪弱”。
- 它不是买点，而是风险保护条件，避免强趋势中因布林带扩张导致 PCTB 暂时回落而过早触发风险。
- 后续验证：`PCTB>0.75` 作为伪弱下限是否过宽或过严。

### PCTB高位惯性

当前讨论状态：

原逻辑：

```pascal
PCTB高位惯性:=趋势健康
AND 价格继续强
AND 布林跟随扩张
AND PCTB>1
AND NOT(长上)
AND NOT(巨额 AND 明显长上);
```

确认整理为：

```pascal
PCTB高位惯性:=趋势健康
AND 价格继续强
AND 布林跟随扩张
AND PCTB>1
AND NOT(长上);
```

- 原因：`明显长上` 一定属于 `长上`，前面已有 `AND NOT(长上)`，所以 `AND NOT(巨额 AND 明显长上)` 重复。
- 作用：识别价格仍在布林上轨上方运行且趋势健康的高位惯性，用于风险保护。

### 布林滞后保护

当前讨论状态：

```pascal
布林滞后保护:=PCTB伪弱 OR PCTB高位惯性;
```

- 接受保留。
- 作用：合并 PCTB伪弱 与 PCTB高位惯性 两类保护，用于后续风险模块，避免强趋势中因布林带扩张或高位惯性运行被过早提示风险。

### 确认强区

当前讨论状态：

```pascal
确认强区:=PCTB>0.90 OR 布林滞后保护;
```

- 接受保留。
- 作用：判断确认当天是否处于强势区域。
- `PCTB>0.90` 是直接强区；`布林滞后保护` 允许趋势健康但 PCTB 因布林扩张表面回落的情况进入确认强区。
- 后续验证：靠 `布林滞后保护` 而非 `PCTB>0.90` 进入确认强区的确认信号，是否真的有确认价值。

### 确认条件拆分

原公式：

```pascal
确认条件:=C>REF(C,1)
AND C>O
AND 实占>=BP
AND 确认强区
AND C>短线
AND 短向>0
AND NOT(长上)
AND (温额 OR 构上 OR REF(首扩启,1) OR REF(速有效,1) OR REF(蓝有效,1) OR REF(冲有效,1));
```

当前确认整理为：

```pascal
确认基础条件:=C>REF(C,1)
AND C>O
AND 实占>=BP
AND 确认强区
AND C>短线
AND 短向>0
AND (温额 OR 构上);
```

含义：

- 原 `确认条件` 不再作为所有确认信号共用条件照搬。
- 先保留确认K线的基础强势要求：收盘继续上涨、阳线、实体占比达标、处于确认强区、站上短线、短线向上，并且有温额或结构突破支持。
- `NOT(长上)`、启动信号来源、启动后第几天、连续收盘推进等内容，后续分别放到 `确2/确3/确4` 的专用条件中讨论。

原因：

- 用户此前已确认取消 `确1`，并且讨论过 `确2` 中取消 `NOT(长上)`。
- 如果继续让 `确2/确3/确4` 共用原 `确认条件`，`NOT(长上)` 会被隐性带回，和前面讨论结论冲突。
- 拆成基础条件与专用条件后，逻辑更清楚，也便于后续分别评价 `确2/确3/确4`。

### 动态 REF 兼容

原来：

```pascal
前5高点距:=HHVBARS(REF(H,1),5)+1;
前5高点PCTB:=REF(PCTB,前5高点距);
```

改为固定展开：

```pascal
前5高点距0:=HHVBARS(REF(H,1),5);

前5高点PCTB:=IF(前5高点距0=0,REF(PCTB,1),
IF(前5高点距0=1,REF(PCTB,2),
IF(前5高点距0=2,REF(PCTB,3),
IF(前5高点距0=3,REF(PCTB,4),REF(PCTB,5)))));
```

影响：

- 交易逻辑不变。
- 提高东方财富通兼容性。

## 确认信号调整

### 取消确1

已确认取消启动后第1天的确认。

### 确2

新版 `确2`：

```pascal
高质启动:=首扩启 OR 速有效 OR 蓝有效 OR 冲有效;

确2启动实体高:=MAX(REF(C,2),REF(O,2));
确2昨日实体高:=MAX(REF(C,1),REF(O,1));
确2前两实体高:=MAX(确2启动实体高,确2昨日实体高);

确2:=REF(高质启动,2)
AND REF(C,1)>REF(C,2)
AND C>确2前两实体高
AND 确认基础条件;
```

含义：

- 前天出现高质量启动。
- 昨天收盘高于启动当天收盘。
- 今天收盘高于启动当天和昨天这两根K线的实体高点。
- 今天同时满足 `确认基础条件`，也就是阳线、实体占比达标、确认强区、站上短线、短线向上，并且有温额或构上支持。
- `确2` 中不加入 `NOT(长上)`。
- 后续验证副图增加 `确2长上警:=确2 AND 长上;`，用于提示确认当天存在长上影瑕疵，但不直接取消 `确2`。

影响：

- `确2` 更少但含义更清楚。
- 主要用于高质启动后的持有或加仓确认。
- 普通黄信号不进入高质启动。
- 新增 `C>确2前两实体高` 后，`确2` 不只是连续收盘推进，还要求确认日收盘突破前两根实体压力。
- 不直接用 `NOT(长上)` 过滤，可以避免误伤强势换手后的确认；但需要观察 `确2长上警` 出现时后续表现是否偏弱。
- `确2` 的量能/结构口径保留 `温额 OR 构上`，不升级为 `放额 OR 构上`。原因是 `确2` 已经有高质启动、连续推进、实体高突破和确认基础条件多层约束，若再强制放额，可能误伤启动后自然缩量但价格继续强推的确认。
- 后续验证副图增加 `确2放额:=确2 AND 放额;`、`确2温额非放:=确2 AND 温额 AND NOT(放额);`、`确2构上:=确2 AND 构上;`，用于比较不同量能来源下的确认质量。
- `确2` 暂不要求启动后到确认前曾经出现过 `PCTB>0.90`。原因是价格可能强势推进且布林带同步扩张，使 `PCTB` 没明显冲到0.90以上；此时如果有布林滞后保护，不应提前排除。
- 后续验证副图增加 `确2前曾强区:=REF(PCTB,1)>0.90 OR REF(PCTB,2)>0.90;`、`确2靠滞后保护:=确2 AND NOT(PCTB>0.90) AND 布林滞后保护;`，用于观察这类确认是否有效。

### 确3

当前讨论基础：

```pascal
确3前三实体高:=REF(HHV(实体高,3),1);

确3:=REF(高质启动,3)
AND C>确3前三实体高
AND 确认基础条件
AND NOT(REF(确2,1));
```

含义：

- 三天前出现高质量启动。
- `确3前三实体高` 取今天之前三根K线的实体高点最高值，也就是启动日、启动后第1天、启动后第2天的实体压力。
- 今天收盘价要高于这三根K线的实体最高点。
- 今天本身满足确认基础条件。
- 昨天如果已经出现过 `确2`，今天不再重复显示 `确3`。

注意：

- 不使用 `HHV(实体高,3)` 直接比较，因为那会把今天自己的实体高点算进去。若今天是阳线，今天实体高通常等于今天收盘，`C>HHV(实体高,3)` 容易自我抵消。
- 因此必须用 `REF(HHV(实体高,3),1)` 看今天之前的三根K线。
- `确3` 不要求启动后第1天、第2天逐日收盘抬高。它的定位是允许启动后中间两天震荡、消化或横盘，只要第3天收盘突破前三根实体高点，并且当天满足确认基础条件，就可以作为晚一天确认。
- 不加入 `REF(C,2)>REF(C,3) AND REF(C,1)>REF(C,2) AND C>REF(C,1)` 这类连续三天收盘上移要求，避免让 `确3` 和 `确2` 过于相似，失去补充意义。
- `确3` 也不加入 `NOT(长上)`。如果确认当天有长上影，不直接取消 `确3`，而是在验证副图记录 `确3长上警:=确3 AND 长上;`。
- 原因：`确3` 已要求收盘突破前三根实体高点，长上影可能只是强势换手；但因为 `确3` 更晚，长上影也可能代表确认时已有抛压，所以需要单独验证其后续表现。
- `确3` 的量能/结构口径沿用 `确认基础条件` 中的 `温额 OR 构上`，不单独升级为 `放额 OR 构上`。
- 后续验证副图增加 `确3放额:=确3 AND 放额;`、`确3温额非放:=确3 AND 温额 AND NOT(放额);`、`确3构上:=确3 AND 构上;`，用于比较不同量能来源下的确认质量。
- `确3` 暂不要求启动后到确认前曾经出现过 `PCTB>0.90`。后续验证副图增加 `确3前曾强区:=REF(PCTB,1)>0.90 OR REF(PCTB,2)>0.90 OR REF(PCTB,3)>0.90;`、`确3靠滞后保护:=确3 AND NOT(PCTB>0.90) AND 布林滞后保护;`。

### 确4

当前讨论基础：

```pascal
确4前四实体高:=REF(HHV(实体高,4),1);
确4中间未坏:=REF(COUNT(C<MID OR PCTB<0.65,3),1)=0;

确4:=REF(高质启动,4)
AND 确4中间未坏
AND C>确4前四实体高
AND 确认基础条件
AND NOT(长上)
AND NOT(REF(确2,2))
AND NOT(REF(确3,1));
```

含义：

- 四天前出现高质量启动。
- `确4前四实体高` 取今天之前四根K线的实体高点最高值，也就是启动日到启动后第3天的实体压力。
- `确4中间未坏` 要求今天之前三根K线里，没有出现收盘跌破布林中轨，或者 `PCTB<0.65` 的明显转弱。
- 今天收盘价要高于这四根K线的实体最高点。
- 今天满足确认基础条件。
- 今天不能有长上影。
- 前天如果已经出现过 `确2`，或者昨天已经出现过 `确3`，今天不重复显示 `确4`。

注意：

- 不使用 `HHV(实体高,4)` 直接比较，因为那会把今天自己的实体高点算进去。
- 因此必须用 `REF(HHV(实体高,4),1)` 看今天之前的四根K线。
- `确4` 比 `确2/确3` 更晚，因此增加 `确4中间未坏`，防止中间已经明显走弱后，今天一根反抽被误判成确认。
- `确4` 正式信号加入 `NOT(长上)`。原因是第4天确认已经偏晚，若确认当天仍出现长上影，说明确认来得晚且当天有冲高回落瑕疵，风险比 `确2/确3` 更高。
- 后续验证副图增加 `确4长上候选:=REF(高质启动,4) AND 确4中间未坏 AND C>确4前四实体高 AND 确认基础条件 AND 长上;`，用于观察被长上过滤掉的候选后续是否确实较弱。
- `确4` 的量能/结构口径沿用 `确认基础条件` 中的 `温额 OR 构上`，不单独升级为 `放额 OR 构上`。
- 后续验证副图增加 `确4放额:=确4 AND 放额;`、`确4温额非放:=确4 AND 温额 AND NOT(放额);`、`确4构上:=确4 AND 构上;`，用于比较不同量能来源下的确认质量。
- `确4` 暂不要求启动后到确认前曾经出现过 `PCTB>0.90`。后续验证副图增加 `确4前曾强区:=REF(PCTB,1)>0.90 OR REF(PCTB,2)>0.90 OR REF(PCTB,3)>0.90 OR REF(PCTB,4)>0.90;`、`确4靠滞后保护:=确4 AND NOT(PCTB>0.90) AND 布林滞后保护;`。

### 险反保留

保留：

```pascal
风险上破:=暴上险 OR 极上险;

险反:=REF(风险上破,1)
AND C>REF(H,1)
AND 阳强
AND NOT(长上)
AND (温额 OR 构上);
```

含义：

- 昨天是风险上破。
- 今天强阳收盘突破昨天最高价。
- 说明昨天风险可能被强势反包。
- `险反` 保留原逻辑，不放宽到最近2日风险。原因是 `C>REF(H,1)`、`阳强`、`NOT(长上)` 已经要求很强，适合作为昨天风险被今天强势否定的反包确认；若放宽到最近2日风险，可能把风险后的普通反抽也纳入。
- 后续验证副图增加 `近2日风险反包候选`，用于观察前天风险、昨天整理、今天再反包的情况是否有必要未来补充。

### 确认最终合成

确认修改为：

```pascal
确:=确2 OR 确3 OR 确4 OR 险反;
```

含义：

- `确1` 已取消，不再参与确认。
- `确2` 表示启动后第2天确认。
- `确3` 表示启动后第3天确认。
- `确4` 表示启动后第4天确认，且比 `确2/确3` 更严格。
- `险反` 表示风险上破后的强势反包确认。

影响：

- 确认模块从原来的 `确1/确2/险反`，改为 `确2/确3/确4/险反`。
- 普通黄信号暂不进入 `高质启动`，后续需要单独讨论普通黄后的确认是否需要独立规则。

### 普通黄后的确认

当前结论：

- 普通黄后的确认暂不加入主公式。
- 原因：普通黄本身质量分层弱于首扩、速、蓝、冲，如果直接纳入 `确`，可能让确认信号变得过泛。
- 后续可以设计 `黄后确认候选`，先进验证副图和回测观察，不进入当前主公式确认模块。

## 风险提示模块

### 续热险前置：已有上轨行为

原公式：

```pascal
已有上轨行为:=NOT(首扩未走出) OR NOT(首扩周期许可);
```

含义：

- `NOT(首扩未走出)` 表示前面已经走出过一段上轨行为，不再是最早期。
- `NOT(首扩周期许可)` 表示当前不再符合新一轮首扩许可，可能已经有过一轮启动或还没有完成重置。
- 合起来表示：只要不是干净的新首扩早期，就认为已经存在上轨运行背景。

当前结论：

- 暂时保留。
- 该变量用于风险模块是合理的，因为风险提示不应过早压制首扩初期，而应更多关注已经跑出一段后的续热阶段。
- 但它是反向推导出来的，不是直接统计价格是否实际在上轨附近运行，因此是重要验证点。

后续验证副图：

```pascal
上轨实际运行数:=COUNT(PCTB>0.85 AND C>短线,黄色趋势段N);
```

用于比较 `已有上轨行为` 与实际价格上轨运行是否一致。

### 续热险

原公式：

```pascal
续热险:=已有上轨行为
AND 带宽初扩
AND PCTB>=1.15
AND PCTB<1+深限
AND (长上 OR 巨额);
```

已讨论部分：

- `已有上轨行为` 暂时保留，作为续热风险的前置环境。
- `带宽初扩` 暂时保留。它表示价格偏热的同时，布林带也开始扩张；用于避免单纯贴近上轨但带宽没有扩张时过早提示续热风险。
- `PCTB>=1.15 AND PCTB<1+深限` 暂时保留。它表示续热险只抓明显偏热、但尚未进入极上区间的阶段。
- `(长上 OR 巨额)` 暂时保留。长上影直接提示冲高回落风险；巨额表示续热阶段分歧或换手异常放大。

潜在问题：

- `带宽初扩` 原本偏启动判断，用在风险里属于借用。它可能把健康早期扩张也纳入续热风险。
- 也可能漏掉已经高宽但扩张速度放缓的续热风险。
- 由于后面还有 `PCTB>=1.15` 和 `长上 OR 巨额`，暂不单独修改。
- 不同周期的续热险 PCTB 区间不同：日线约为 `1.15-1.30`，周线约为 `1.15-1.25`，月线约为 `1.15-1.20`。月线区间较窄，可能导致月线续热险较少；日线 `1.15` 可能偏敏感。
- `巨额` 单独触发续热险可能偏敏感。强势主升中，巨额有时代表资金确认突破，不一定是风险。暂不改为 `长上 OR (巨额 AND PCTB>=1.20)`，先通过验证判断。

后续验证：

- 验证副图需对照显示 `带宽初扩` 与 `高宽风险环境`，观察 `续热险` 到底更常出现在早扩阶段还是高宽风险阶段。
- 后续验证 `续热险` 在月线是否过少，以及日线 `PCTB>=1.15` 是否太敏感。
- 验证副图拆分 `续热长上源:=续热险 AND 长上;`、`续热巨额源:=续热险 AND 巨额;`、`续热纯巨额源:=续热险 AND 巨额 AND NOT(长上);`，重点观察单纯巨额触发的续热险后续到底是风险还是强势延续。

### 高宽风险环境参数

原公式：

```pascal
高宽N:=IF(周线,5,IF(月线,3,10));
高宽阈:=IF(周线,3,IF(月线,2,5));
```

含义：

- 日线：最近10天里至少5天高宽，才算持续高宽。
- 周线：最近5周里至少3周高宽。
- 月线：最近3个月里至少2个月高宽。

当前结论：

- 暂时保留。
- 这组参数用于判断高宽是否具有持续性，不因单根K线高宽就进入高宽风险环境。
- 日线 `10/5` 比较中性；周线 `5/3` 偏稳；月线 `3/2` 样本较少，可能有跳变。

后续验证：

- 验证副图需显示 `宽带绝对持续高`、`宽带相对持续高`、`宽带持续高`、`高宽风险环境`。
- 观察高宽风险环境成立后，后续风险提示是否有效。

### 宽带持续高

原公式：

```pascal
宽带绝对持续高:=COUNT(带宽>带宽绝高线,高宽N)>=高宽阈;
宽带相对持续高:=COUNT(带宽/MAX(有效宽均,0.0001)>带宽相高线,高宽N)>=高宽阈;

宽带持续高:=宽带绝对持续高 OR 宽带相对持续高;
```

含义：

- `宽带绝对持续高`：最近一段时间内，带宽多次超过固定绝对高宽线。
- `宽带相对持续高`：最近一段时间内，带宽多次超过自身历史平均带宽的一定倍数。
- `宽带持续高`：绝对路径或相对路径任意一条成立，就认为带宽持续高。

当前结论：

- 保留，不改。
- 该结构同时照顾波动本来就大的股票，以及平时波动不大但突然明显放大的股票。
- `OR` 关系可能偏敏感，尤其低波动股票的相对高宽路径可能较容易成立，但作为风险环境可先保留。

后续验证：

- 验证副图必须拆分显示 `宽带绝对持续高`、`宽带相对持续高`、`宽带持续高`。
- 如果后续高宽风险太敏感，需要判断主要是绝对路径还是相对路径导致。

### 高宽风险环境

原公式：

```pascal
高宽风险环境:=宽带持续高 AND NOT(扩张保护期);
```

含义：

- `宽带持续高` 表示最近一段时间带宽已经持续处于高位。
- `扩张保护期` 表示刚从收缩状态启动扩张、趋势仍健康的早期保护阶段。
- 两者合起来表示：带宽持续高，而且当前不在健康启动后的扩张保护期，才认为进入高宽风险环境。

当前结论：

- 保留，不改。
- 该逻辑用于避免刚启动、刚扩张时因为带宽变高而过早提示风险。
- 真正需要重点验证的是 `扩张保护期` 是否定义合适。

潜在问题：

- 如果 `扩张保护期` 定义过宽，可能把已经有风险的高宽环境保护掉。
- 如果 `扩张保护期` 定义过窄，可能刚启动没多久就进入高宽风险环境。

### 扩张保护期

原公式：

```pascal
保护N:=IF(周线,3,IF(月线,2,8));

扩张保护起点:=真正收缩
AND 带宽初扩
AND PCTB>0.70
AND PCTB<1+深限
AND 上轨趋势
AND NOT(巨额 AND 长上);

扩张保护期:=BARSLAST(扩张保护起点)<=保护N
AND PCTB>0.60
AND C>短线;
```

含义：

- `保护N` 是扩张保护期长度：日线8根、周线3根、月线2根。
- `扩张保护起点` 表示真正收缩后，带宽刚初扩，价格处在偏强区间，趋势环境向上，且没有巨额长上影。
- `扩张保护期` 表示距离扩张保护起点还不久，并且当前仍满足 `PCTB>0.60`、收盘价站在短线上方。

当前结论：

- 暂时保留，不改主公式。
- 这层保护用于避免刚启动、刚扩张时过早进入高宽风险环境。
- 日线8根保护略长，但先不直接缩短；后续通过验证判断是否导致风险提示偏晚。

潜在问题：

- `PCTB>0.60` 较宽，可能保护一些已经明显从上轨回落、但还没跌破短线的走势。
- `C>短线` 是必要的基础保护条件，但无法识别保护期内的放量滞涨、长上影、收阴弱化等细节。

后续验证：

```pascal
距扩张保护起点:=BARSLAST(扩张保护起点);

保护期弱化候选:=扩张保护期
AND (长上 OR C<REF(C,1) OR 滞源 OR 慎源);
```

验证目的：

- 检查 `扩张保护期` 是否过宽或过窄。
- 观察保护期内出现长上影、收盘低于昨日、慎/滞迹象时，后续是否更容易走弱。
- 如果这类弱化候选后续风险明显增加，未来再考虑缩短保护或解除保护。

## 挤压模块

### 挤压定义

保持：

```pascal
挤压:=BUP<KCU AND BDN>KCD;
```

含义：

- 布林带完全缩进 Keltner 通道。

### 近挤

原来：

```pascal
近挤:=COUNT(挤压,SQN)>=1;
```

改为：

```pascal
近挤:=COUNT(挤压,2)>=1;
```

影响：

- 近挤只表示最近2根K线内有挤压。
- 主要负责“刚刚挤压后的快速突破背景”。
- 延迟突破由迟发挤压背景补充。

### SQN 与效挤

新版：

```pascal
SQN:=IF(周线,3,IF(月线,3,5));

效挤:=IF(日线,
COUNT(挤压,5)>=2 AND BARSLAST(挤压)<=2,
COUNT(挤压,SQN)>=2);
```

含义：

- 日线：最近5天至少2次挤压，且最近一次在今天、昨天或前天。
- 周线：最近3周至少2次挤压。
- 月线：最近3个月至少2次挤压。

影响：

- 日线效挤更强调新鲜度。
- 月线从2个月内2次改成3个月内2次，更符合月线节奏。
- 效挤本身不加带宽未扩条件，后续在有效挤压合成时统一加宽度许可。

### 分散挤原

保留作为有效挤压的第二类挤压来源。

新版确认：

```pascal
日分散挤原:=COUNT(挤压,10)>=2 AND COUNT(挤压,3)>=1;
周分散挤原:=COUNT(挤压,7)>=2 AND COUNT(挤压,2)>=1;
月分散挤原:=COUNT(挤压,5)>=2 AND COUNT(挤压,2)>=1;

分散挤原:=IF(日线,日分散挤原,IF(周线,周分散挤原,月分散挤原));
```

含义：

- 日线：最近10天内至少2次挤压，并且最近3天内至少1次挤压。
- 周线：最近7周内至少2次挤压，并且最近2周内至少1次挤压。
- 月线：最近5个月内至少2次挤压，并且最近2个月内至少1次挤压。
- 不再要求两次挤压相隔固定距离，因为后续还有 `挤压宽许可` 控制质量。

影响：

- 分散挤原更偏向识别“中等窗口内反复挤压且近期仍有挤压”的背景。
- 相比固定距离版本，条件更简洁，也避免为了间隔距离漏掉合理的反复挤压。
- 由于后续合成 `有效挤压` 时仍要求 `挤压宽许可`，所以不会单独因为分散挤原略放宽就直接放行高宽背景。

### 挤压宽许可

新增：

```pascal
挤压宽许可:=带宽/MAX(前有效宽均,0.0001)<1.15
AND NOT(启动前持宽)
AND NOT(REF(宽近高,1));
```

含义：

- 当前带宽不能超过前有效均宽的1.15倍。
- 启动前不能已经持宽。
- 上一根不能已经接近阶段高宽。

### 有效挤压

原来：

```pascal
有效挤压:=REF(效挤,1) OR REF(COUNT(挤压,SQN*2),1)>=1;
```

新版确认：

```pascal
有效挤压:=(REF(效挤,1) OR 分散挤原) AND 挤压宽许可;
```

影响：

- 原来较长窗口内1次挤压就能放行，偏宽。
- 新版要求密集挤压或分散反复挤压，同时带宽仍未明显扩张。
- 有效挤压更干净，避免旧挤压或偶然挤压直接升级为真正收缩。

## 迟发挤压背景

### 用途

迟发挤压背景只用于突破背景，不直接放入有效挤压。

用途：

```pascal
突破背景:=REF(近挤,1)
OR 迟发挤压背景
OR REF(真正收缩,1)
OR 真正收缩;
```

含义：

- `REF(近挤,1)` 抓快速突破。
- `迟发挤压背景` 抓挤压后延迟突破但带宽未明显扩张。
- `真正收缩` 保留原有收缩突破路径。

### 迟发窗口

```pascal
迟发挤N:=IF(周线,6,IF(月线,6,10));

迟发曾挤:=REF(COUNT(挤压,迟发挤N),1)>=1;
```

含义：

- 日线：昨天往前10天内至少1次挤压。
- 周线：上周往前6周内至少1次挤压。
- 月线：上月往前6个月内至少1次挤压。

### 2年位置分档

```pascal
位N:=IF(周线,104,IF(月线,24,500));
位高:=HHV(H,位N);
位低:=LLV(L,位N);
二年位:=(C-位低)/MAX(位高-位低,0.0001);

高位2年:=二年位>=0.80;
低位2年:=二年位<=0.40;
```

含义：

- 高位更严格。
- 低位略宽。

### 迟发绝宽线

```pascal
迟发绝宽线:=IF(高位2年,
IF(周线,0.35,IF(月线,0.45,0.22)),
IF(低位2年,
IF(周线,0.45,IF(月线,0.55,0.28)),
IF(周线,0.40,IF(月线,0.50,0.25))));

迟发绝宽许可:=带宽<迟发绝宽线;
```

含义：

- 高位：日0.22、周0.35、月0.45。
- 中位：日0.25、周0.40、月0.50。
- 低位：日0.28、周0.45、月0.55。
- 只要求当前K线带宽低于迟发绝宽线，不强制上一根也低于。

### 迟发近宽未破

```pascal
迟发近宽N:=IF(周线,3,IF(月线,3,5));
迟发近宽未破:=带宽<=REF(HHV(带宽,迟发近宽N),1)*1.03;
```

含义：

- 日线：当前带宽不能超过前5天最高带宽的1.03倍。
- 周线：不能超过前3周最高带宽的1.03倍。
- 月线：不能超过前3个月最高带宽的1.03倍。
- 不再用整个迟发窗口的最高带宽，避免被窗口早期高宽误导。

### 迟发宽速许可

```pascal
迟发宽速许可:=宽速率<=微扩阈;
```

含义：

- 日线：当前带宽相对5天前扩张不超过4%。
- 周线：相对3周前扩张不超过5%。
- 月线：相对2个月前扩张不超过6%。
- 允许负数，带宽收缩也通过。

### 迟发宽未扩

```pascal
迟发宽未扩:=迟发绝宽许可
AND 迟发近宽未破
AND 带宽/MAX(前有效宽均,0.0001)<1.08
AND NOT(启动前持宽)
AND NOT(REF(宽近高,1))
AND 迟发宽速许可;
```

含义：

- 当前带宽低于迟发绝宽线。
- 当前带宽没有突破近窗口高宽。
- 当前带宽低于前有效均宽1.08倍。
- 启动前没有持宽。
- 上一根没有接近阶段高宽。
- 当前带宽相对前一段没有明显扩张。

### 迟发挤压背景

```pascal
迟发挤压背景:=迟发曾挤 AND 迟发宽未扩;
```

含义：

- 前面曾经有挤压。
- 但当前带宽仍没有明显持续扩张。
- 用于补充延迟突破背景。

## 启动前持宽

已确认结构：

```pascal
硬高宽:=带宽绝高 OR 前带宽绝高 OR 带宽相高;
软持宽:=宽近高 AND 宽速率<=微扩阈;

持宽:=硬高宽 OR 软持宽;
启动前持宽:=REF(持宽,1);
```

含义：

- 硬高宽：当前或上一根绝对高宽，或当前/上一根相对均宽明显偏高。
- 软持宽：当前接近阶段高宽，并且没有明显继续扩张。
- 启动前持宽：上一根K线已经处于持宽状态。

## 后续验证指标构想

用户提出：后续希望建立一个单独副图验证指标，用于快速诊断单根K线的信号成因。

目标：

- 鼠标指向某根K线时，可以快速看到该K线的核心判断信息。
- 用于解释“为什么这里出信号”或“为什么这里没出信号”。
- 避免后续每次让模型重新验证大量条件，提升定位效率。
- 更重要的目标：提供全面的底层状态信息，用来判断某处“出信号是否合理”以及“没出信号是否合理”，从而反向校准主公式的逻辑和参数。
- 验证指标不仅服务于解释结果，也服务于发现公式缺陷，例如挤压背景是否遗漏重要形态、是否错放了不好的背景、顶部风险是否没有被显示、某些阈值是否过严或过松。
- 终极目标是让主公式更合理、更完善，而不是让信号过于密集。

后续验证指标应尽量覆盖以下信息层：

- 基础K线：收盘价、开盘价、最高价、最低价、涨跌、实体长度、实体占比、上影比例、下影比例、是否阳强、是否长上、是否中阳无长上。
- 量能：成交额、均额、温额、放额、巨额、缩额、有额。
- 布林/PCTB：MID、BUP、BDN、PCTB、是否上穿1、是否极上、是否处于确认强区。
