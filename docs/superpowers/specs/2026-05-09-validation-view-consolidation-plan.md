# 验证视图合并计划

## 概览

东方财富通时代为了绕过公式长度和后置绘图不稳定，只能把 A 图拆成多张短副图。本地公式验证实验室没有这个限制，因此不再按旧副图 1:1 复刻，而是改成“少量验证视图 + 点击单根 K 线看条件”的结构。

## 设计原则

- 主图保持清爽，只保留 K 线、BOLL、成交量、PCTB。
- 右侧面板按视图切换，不同时铺满所有字段。
- 每个视图控制在 10 个以内字段，超过就继续拆分或放到详情层。
- 旧东方财富通副图暂不删除，作为口径对照和历史记录。
- 每搬一小组变量，就立即接入视图并测试，不做一次性大迁移。

## 旧副图合并映射

| 本地视图 | 合并旧副图 | 用途 |
|---|---|---|
| 总览 | A1 / A2 / A3 关键结论 | 一眼判断这根 K 卡在基础、背景、上穿还是质量层。 |
| 基础 | A1-clean / A1-front | K 线质量、上影、实体、收盘位置、量能和结构突破。 |
| 背景 | A2-readable / A3a2 / A3b2b1 | 挤压、收缩、迟发、宽度挡、旧/新突破背景。 |
| 上穿来源 | A3a1 / A3a3 / A3b2a 当前已搬部分 | 上穿、浅绕过滤、温上、暴上、极上和风险源。 |

## 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 字段太多导致右侧面板拥挤 | 本地实验室变成另一种副图堆 | 每个视图最多 10 个字段，必要时拆二级详情。 |
| Python 移植与东方财富通语义有差异 | 断点结论不一致 | 用典型样本逐项交叉确认，差异优先记录为平台语义问题。 |
| 旧副图仍有文档引用 | 误删历史依据 | 暂不删除旧文件，只在索引中标注本地实验室为主路径。 |
