# 本地公式验证实验室实现计划

## 概览

按已确认设计，第一阶段先实现“数据下载冒烟”这一条最小闭环：从股票池读取股票，使用 AKShare 下载前复权日线，标准化字段后缓存到本地。通过 `002222 福晶科技` 冒烟测试确认数据源、复权口径和缓存结构可用。冒烟成功后，继续接一个最小可用浏览器界面，用于验证 K 线、BOLL 和 PCTB 副图能脱离东方财富通显示。

## 架构决定

- Python 先承担数据层和本地 HTTP 服务。
- 第一版前端使用 ECharts，通过本地 API 读取数据和指标。
- 第一版主数据源是 AKShare `stock_zh_a_hist`。
- 第一版历史价格口径固定为 `adjust="qfq"` 前复权。
- 缓存数据不进入 Git。
- 先写可测试的小函数，再接外部下载。

## 任务列表

### 任务 1：数据层基础

描述：建立 Python 包、依赖文件、股票池读取、股票代码转换和缓存目录规则。

验收标准：

- 能把 `002222` 转成 AKShare 所需代码 `002222`，并保留未来转换到 `.SZ/.SH/.BJ` 的能力。
- 能从 Excel 股票池读取代码和名称。
- 数据缓存目录不进入 Git。

验证：

- 运行 `python -m unittest discover tests`。

### 任务 2：AKShare 冒烟下载

描述：实现一只股票的前复权日线下载脚本，并缓存为 CSV。

验收标准：

- 能下载 `002222` 从 `20240101` 到当前日期的前复权日线。
- 输出字段包含日期、开盘、最高、最低、收盘、成交量、成交额、涨跌幅、换手率。
- 缓存文件写入 `data/raw/akshare/qfq/daily/`。

验证：

- 运行冒烟脚本并检查输出行数、日期范围和字段。

### 任务 3：指标计算起步

描述：实现 BOLL 和 PCTB 的基础计算，为 A3b2b1 诊断做准备。

验收标准：

- 能对冒烟数据计算 BOLL(20,2)。
- 能计算 PCTB。
- 能输出带指标的 CSV 或 JSON。

验证：

- 对已缓存的 `002222` 数据运行指标脚本。

### 任务 4：最小可用界面

描述：启动本地服务，提供 K 线、BOLL、成交量和 PCTB 副图。

验收标准：

- 能在浏览器打开 `http://127.0.0.1:52410/`。
- 页面能加载 `002222` 前复权日线数据。
- 最新一根 K 线的 BOLL 与东方财富通样本一致。

验证：

- 调用 `/api/health`。
- 调用 `/api/daily?code=002222&start=20240101&end=20260509`。
- 使用 Playwright 截图确认页面非空且图表渲染。

### 阶段检查点

- 数据下载成功。
- 字段和单位明确。
- 前复权口径可接受。
- 浏览器界面雏形可打开。

## 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| AKShare 接口波动 | 下载失败 | 加重试和错误日志，允许手动导入兜底 |
| 字段单位与东方财富界面不一致 | 指标计算偏差 | 冒烟阶段先核对成交量、成交额、价格口径 |
| 前复权价格与界面略有差异 | 诊断偏差 | 保留数据源标记，后续可接 Tushare 或手动导出交叉校验 |
| 数据量扩大后变慢 | 使用体验差 | 第一阶段只跑样本池，后续再做增量更新 |
