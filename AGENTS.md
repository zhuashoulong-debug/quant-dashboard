# AGENTS.md

本仓库用于量化 / 回测 / 策略看板相关开发。Codex 或其他 AI 编程助手修改本仓库时，请遵守以下规则。

## 工作原则

1. 优先做小步修改，避免一次性大改多个无关模块。
2. 修改前先理解现有文件结构和变量命名，不要随意重命名核心变量。
3. 涉及交易信号、买卖规则、仓位规则、回测逻辑时，必须保持逻辑可追溯、可验证。
4. 不要把诊断代码混入正式策略逻辑；如需验证，应单独新增诊断脚本、诊断输出或副图/辅助模块。
5. 不要提交密钥、Token、账号、密码、券商凭证、真实交易接口凭证等敏感信息。

## 代码修改要求

1. 修改某个文件时，尽量保持原有风格、目录结构和依赖方式。
2. 涉及公式、指标、策略规则时，优先给出完整可替换代码块，不要只给零散片段。
3. 涉及回测结果时，尽量输出关键中间事件，方便排查信号是否按预期触发。
4. 若新增输出文件，优先放在 `bt_output/` 或项目已有输出目录下。
5. 若项目中已有测试、lint、format 命令，修改后应说明是否运行，以及结果如何。

## 投资与回测注意事项

1. 回测结果只用于研究，不构成投资建议。
2. 策略修改后需要重新回测，并在实盘前做小样本校准。
3. 对影响较大的策略参数，应尽量保留对比实验入口，避免直接覆盖旧逻辑。

## 回复偏好

1. 默认使用中文说明修改内容。
2. 说明问题时优先采用“这个条件回答什么问题 / 不回答什么问题”的结构。
3. 给出结论时要区分：已经确认、合理推断、仍需验证。
