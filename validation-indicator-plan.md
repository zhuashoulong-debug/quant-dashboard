# 验证/诊断副图设计计划

本文件用于后续设计验证副图。验证副图不是交易信号本身，而是公式解释器和校准仪表盘。

## 目标

- 判断某根K线为什么出信号。
- 判断某根K线为什么没出信号。
- 判断信号成立是否合理。
- 判断信号不成立是否合理。
- 辅助发现主公式逻辑或参数是否需要调整。

## 当前状态

等待用户说“开始验证指标讨论”后正式展开。

## 实现优先级

用户已确认验证副图实现顺序为：

```text
A → B → C → D
```

含义：

1. A 图：基础 / 背景 / 源头；
2. B 图：过滤 / 冷却 / 显示；
3. C 图：确认 / 风险 / 慎滞；
4. D 图：新旧公式对照 / 未显示原因。

采用这个顺序的原因：

- 先用 A 图判断是否具备潜在信号基础；
- 再用 B 图解释源头成立后为什么没显示；
- 接着用 C 图处理确认、风险、慎滞这些关键判断；
- 最后用 D 图比较新旧公式差异和最终未显示原因。

## 字段审计机制

后续每讨论一个模块，都要同步问三件事：

1. 这个模块的最终判断需要哪些底层变量支持；
2. 如果信号成立/不成立，哪些变量必须显示出来才能解释原因；
3. 如果以后要微调参数，哪些中间变量必须显示出来才能判断调得是否合理。

每完成一个大模块后，需要做一次字段审计：确认该模块的源头条件、过滤条件、保护条件、风险条件、显示优先级，都能在验证副图里被追踪。

最终整理公式前，还要按整理版公式逐行反查：凡是影响最终信号、风险、确认、慎滞、失败的变量，都必须能在验证副图或条件断点表里找到对应字段。

验证副图字段设计阶段宁可多做字段审计，也不能粗略跳过。原因是：

- 字段漏掉，会导致后续无法快速解释某根K线为什么出信号或不出信号；
- 字段漏掉，会导致无法判断信号成立是否合理；
- 字段漏掉，会导致参数微调缺乏依据；
- 字段漏掉，会让诊断重新回到漫长、分散、逐条猜测的状态。

每完成一个验证副图区块，都要主动做补漏检查：

- 是否覆盖底层变量；
- 是否覆盖动态阈值；
- 是否覆盖源头、过滤、保护、显示优先级；
- 是否覆盖新旧公式差异；
- 是否覆盖后续参数微调需要依赖的中间变量。

## 信息不足快速判定

诊断某根K线时，如果出现以下情况，不能继续凭感觉硬查：

- 只要有一个决定性变量缺失，立即判定为“当前副图对该问题信息不足”；
- 如果缺的是辅助变量，可以先继续判断，但必须记录为待补字段；
- 如果因为缺字段导致无法判断信号成立/不成立是否合理，应立即停止硬查，进入缺口分类；
- 不允许为了继续回答而凭感觉猜。

决定性变量指：没有它，就无法判断某个关键条件为什么成立或失败。

示例：

- 不知道为什么 `温上` 没成立，如果缺 `有效上穿1` 或 `突破背景`，就是决定性变量缺失；
- 不知道为什么 `迟发挤压背景` 没成立，如果缺 `迟发宽未扩` 或其子条件，就是决定性变量缺失；
- 不知道为什么风险没显示，如果缺 `普通险原`、`不可早期过滤硬险` 或 `初扩早期险过滤`，就是决定性变量缺失。

缺口分类：

- 基础形态缺口；
- 成交额缺口；
- 价格位置缺口；
- 带宽缺口；
- 挤压/迟发缺口；
- 收缩缺口；
- 源头缺口；
- 显示/过滤缺口；
- 风险缺口。

## 主公式与验证副图同步原则

长期锚点：

- 后续主公式做任何调整时，必须同步检查所有验证副图是否需要更新；
- 验证副图必须尽量保持和主公式同口径；
- 不能让验证副图用旧变量、旧阈值、旧显示优先级去解释新公式。

每次主公式调整后，需要检查：

- A 图：基础 / 背景 / 源头；
- B 图：过滤 / 冷却 / 显示；
- C 图：确认 / 风险 / 慎滞；
- D 图：新旧公式对照 / 未显示原因；
- 所有当前仍被采用、有效、最新定稿或准定稿的验证副图版本，不限于 `mini/debug/readable` 这几个命名；
- 回测字段与对比表；
- 文件中的中文读法说明。

如果某张副图暂时没有同步更新，必须在文件说明或当前断点中标注：该副图口径已落后于主公式，不能用于最终判断。
