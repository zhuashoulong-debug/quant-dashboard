# B 卖点 v13 下一轮主公式调整候选

更新时间：2026-05-10

用途：承接最新 200 样本复核结果，整理下一轮是否修改主公式的候选方向。当前只列候选和判断状态，不直接改主公式。

## 总原则

- [已确认] 200 样本不支持“主公式整体过敏”。
- [暂不改待验证] 在核图 `弱 / 形警 / 天` 之前，主公式 v13 不做新改动。
- [已确认] 简单单阈值过滤暂时不成立：成交额比例、MA20 破位质量、MA5 下压都不能稳定区分敏感与有效样本。
- [记录观察] 下一步要找的是“过程结构保护”，不是硬加一个孤立阈值。

## 候选一：收紧 `弱`

用途：减少 MA20 结构转弱后的快速修复样本。

当前观察：

- [记录观察] `弱` 是非确认段最大来源，200 样本里非确认段 16 个。
- [已确认] 但 `弱` 的有效事件仍有 381 个，整体有效率高。
- [已确认] 非确认 `弱` 并不只是浅破；破位质量条件已经成立。
- [暂不改待验证] 不建议直接提高现有 MA20 破位质量阈值。

候选方向：

- [只进验证副图] 观察 `弱` 后 1-3 日是否快速收回 MA20。
- [只进验证副图] 观察 `弱` 当天是否仍在强上升通道内，例如 MA20 仍明显上行、MA60 上行、收盘未破关键 K 中位。
- [记录观察] 如果核图确认多数 `弱` 是强趋势中的短暂回踩，可以考虑把 `弱` 分成“弱预警”和“弱确认”两档。
- [暂不改待验证] 暂不直接给 `弱` 加成交额过滤，因为有效样本也大量低额。

## 候选二：处理 `形警`

用途：减少低层形态提示在强趋势中的轻微打扰。

当前观察：

- [记录观察] `形警` 非确认段 9 个，来源基本是射击星。
- [已确认] `形警` 总体敏感率不高，且它本来就是最低层提示。
- [暂不改待验证] 不建议删除 `形警`。

候选方向：

- [只进验证副图] 观察射击星是否发生在已有高层风险之后；如果是，保留作为低层解释即可。
- [只进验证副图] 观察射击星当天是否收盘仍强、次日是否快速反包。
- [记录观察] 若核图显示多数 `形警` 是强趋势中的普通上影，优先降低显示存在感，而不是改成更重的卖点。
- [暂不改待验证] 暂不把 `形警` 升级为确认层，也不直接删除。

## 候选三：保护 `天`

用途：区分“天量天价顶部异常”和“强趋势换手继续上攻”。

当前观察：

- [记录观察] `天` 非确认段 4 个，样本不多。
- [已确认] `天` 的成交额强度不能区分敏感与有效，敏感事件和有效事件都明显放额。
- [暂不改待验证] 不建议单纯提高天量阈值。

候选方向：

- [只进验证副图] 观察 `天` 当天是否收盘接近最高价、是否上影很短。
- [只进验证副图] 观察 `天` 前 3 日是否已有稳步推进或强趋势换手。
- [记录观察] 如果非确认 `天` 多是强收、短上影、次日继续新高，可以考虑给 `天` 增加“强收保护”。
- [暂不改待验证] 未核图前，不直接修改 `天量天价`。

## 候选四：确认层显示压缩

用途：降低 `强败 / 天败` 在同一风险段里的重复显示密度。

当前观察：

- [已确认] 确认层事件 542 个。
- [记录观察] 同一确认段只保留首次确认的模拟，会从 542 个减少到 436 个。
- [记录观察] 5 日确认层压缩模拟，会从 542 个减少到 473 个。
- [已确认] 这属于读图密度问题，不是卖点有效性问题。

候选方向：

- [只进验证副图] 继续观察 `confirmation_compress_first` 和 `confirmation_compress_5d`。
- [暂不改待验证] 当前不直接改主公式确认层显示。
- [记录观察] 如果东方财富主图实际显示太挤，再讨论确认层压缩是否进入正式公式。

## 候选五：严格动态 REF 兼容版

用途：处理东方财富通可能不支持动态 `REF` 的情况。

当前观察：

- [已确认] 主公式严格版只保留 3 处动态 `REF`，都属于核心严格逻辑。
- [暂不改待验证] 不主动替换为兼容近似版。
- [记录观察] 如果验证副图在动态 `REF` 附近编译失败，再启用兼容版设计。

- [下一步] 本地验证侧：优先核图 `弱 / 形警 / 天` 非确认段。
- [暂不改待验证] 在东方财富编译结果和非确认段核图结果出来前，主公式 v13 保持不变。
